## 1. 执行摘要

“小陪”是一款面向家庭的智能陪伴机器人，基于 RDK X5 平台设计，目标进入中端市场（目标零售价：标准版 2999 元，高级版 3999 元）。产品主打“情感陪伴 + 实用生活助手”两大核心价值，面向独居老人、儿童、年轻上班族与家庭用户，提供情感交互、视觉感知、生活辅助、家庭安防、娱乐互动与家居控制等能力。

本PRD旨在定义产品范围、功能细节、技术实现要点、MVP 边界、测试与合规要求、以及商业化路线，作为研发、硬件、测试、合规与市场团队的工作蓝图。

## 2. 产品概述

### 2.1 产品愿景

打造一个能够持续学习、温暖可靠、可持续升级的家庭陪伴平台，既能在情感层面与用户建立联结，又能在实用功能上真正减少家庭照护与生活安排成本。

### 2.2 主要目标

* 提供常驻家庭的情感陪伴，帮助减轻孤独与社交隔离感
* 保障家庭成员安全（老人跌倒、火灾、异常入侵）
* 帮助家庭完成基础生活事务（用药提醒、行程提醒、家电控制）
* 构建可扩展的生态与增值服务能力

### 2.3 关键约束

* 目标生产成本控制（BOM）以保证零售价竞争力
* 遵循数据隐私法规（PIPL 等）与电磁兼容/安全认证
* 在 RDK X5 等目标硬件上部署 AI 模型，兼顾本地推理与云服务

## 3. 目标用户与用户画像

### 3.1 用户画像（示例）

* **独居老人-张奶奶**：70岁，退休，生活自理能力一般，偶尔忘记吃药。需要有人陪伴、用药提醒与紧急呼叫。
* **家长-李女士**：35岁，上班族，关心孩子课后学习与安全。希望机器人能辅导孩子、播放故事、并在她不在时监护孩子安全。
* **年轻上班族-王先生**：28岁，单身，常出差。希望远程查看家中情况、控制智能家电、与宠物远程互动。
* **家庭-陈氏一家三口**：需要周末家庭娱乐、节日合影与智能场景联动。

### 3.2 用户需求摘要

* 情感陪伴、主动问候与对话
* 及时紧急联动（SOS、跌倒、烟雾）
* 简单易用的交互方式（语音优先 + 触控备用）
* 隐私与数据可控（家长与老人都关注）
* 与现有智能家居生态兼容

## 4. 市场与竞争分析

（略写要点，详见市场研究附件）

* 目标市场：中端家庭陪伴机器人市场，中国 2025 年市场估值（假设）已处于快速增长期。
* 主要竞争者：国际高端（Pepper、NAO）、国内中低端（小度机器人、小米铁蛋、儿童教育机器人）。
* 差异化机会：情感陪伴 + 健康/安全场景深耕；价格与功能的平衡；本地隐私优先策略。

## 5. 产品定位与价值主张

* **定位**：面向中端家庭的智能陪伴机器人，兼顾情感陪伴与实用生活助手。
* **价值主张**：建立“人—机—家庭”的持续陪伴关系，降低孤独与照护成本，提升家庭生活便利性和安全性。

## 6. 功能详述（含用户故事与验收标准）

> 本章列出功能模块、子功能、优先级（P0/P1/P2）、示例用户故事及具体验收标准（Acceptance Criteria）。

### 6.0 优先级说明

* P0：MVP 必须项，必须在第一版本交付
* P1：重要项，第一轮迭代中完成
* P2：后续迭代完善

### 6.1 情感交互模块（P0/P1）

**功能**：

* 语音唤醒与识别（ASR）
* 基础语义理解（NLU）与多轮对话管理
* TTS（本地/云混合）
* 情绪识别（语音情绪 + 面部表情）
* 个性化记忆库（用户档案、偏好、重要日历）

**用户故事**：

* 作为独居老人，张奶奶希望机器人在她起床时主动问候并提醒吃早药。
* 作为家长，我希望机器人记住孩子喜欢的故事类型并在睡前播放。

**验收标准**：

* 语音唤醒误触率 ≤ 3%（典型家庭噪声下）
* ASR 准确率：安静环境 ≥ 92%，一般室内噪声 ≥ 85%
* 长期记忆可查询/删除：用户可通过 App 或语音删除特定记忆

### 6.2 视觉感知模块（P0/P1）

**功能**：

* 人脸识别（多用户）与权限管理
* 姿态识别、跌倒检测
* 场景识别与物体检测（人物、宠物、烟雾异常）
* 跟随与运动控制（保持合理距离）

**用户故事**：

* 若检测到老人跌倒并长时间无响应，自动触发 SOS 并通知家属。

**验收标准**：

* 人脸识别在常见光照下误识率 ≤ 1%
* 跌倒检测召回率 ≥ 90%，误报 ≤ 5%

### 6.3 生活辅助模块（P0/P1）

**功能**：

* 日程与用药提醒（支持重复规则、确认与记录）
* 健康监测数据接入（蓝牙传感器：血压/心率/血氧）并呈现趋势
* 紧急求助（SOS 按钮、语音急救触发、自动拨号）

**用户故事**：

* 家属可在 App 中查看老人连续 7 天的血压曲线与异常提醒。

**验收标准**：

* 用药提醒必须在设定时间发出并记录确认状态
* 第三方蓝牙设备连接成功率 ≥ 95%（兼容清单内设备）

### 6.4 远程监控模块（P0）

**功能**：

* 手机 APP 实时视频与双向语音通话
* 异常报警推送（陌生人、烟雾、漏水）
* 历史录像与事件回放
* 隐私模式（可按时间/区域/触发条件禁用摄像/麦克风）

**用户故事**：

* 王先生出差时希望能远程与家中宠物说话并查看实时画面。

**验收标准**：

* 局域网内视频延时 < 300 ms（30 fps，自适应码率）
* 云推送事件到 App 的平均延时 < 10 s（网络正常时）

### 6.5 娱乐互动模块（P1）

**功能**：

* 儿童故事、课后辅导、互动游戏
* 音乐/电台流媒体播放（本地 + 云）
* 节假日主题与家庭合影功能

**用户故事**：

* 家长可查看孩子的学习进度并获得推荐练习题。

**验收标准**：

* 儿童内容通过家长后台管理并支持黑名单/白名单控制

### 6.6 家居控制模块（P1）

**功能**：

* 支持 Wi-Fi、Bluetooth LE、Zigbee、IR、Matter 协议
* 场景模式编辑器（回家/离家/睡眠/观影）
* 环境监测（温湿度、PM2.5）与自动动作触发

**用户故事**：

* 回家场景：机器人检测到主人回家自动开灯、播放音乐、播报天气。

**验收标准**：

* 本地控制响应时间 ≤ 2 s
* 场景联动稳定执行率 ≥ 98%

### 6.7 设备管理与 OTA（P0）

**功能**：

* 安全的 OTA 管理（固件签名、分段更新、回滚）
* 设备健康状态上报（心跳、关键资源使用）
* 远程日志抓取与诊断工具

**验收标准**：

* OTA 更新成功率 ≥ 99%（网络可用时）
* 回滚可在更新失败后自动触发

## 7. 技术与硬件架构

### 7.1 硬件选型（基于 RDK X5）

* **主控**：RDK X5，8 核 Cortex-A55 @1.5GHz
* **AI 加速**：10 TOPS BPU
* **内存**：8GB LPDDR4
* **存储**：128GB eMMC，支持 MicroSD
* **摄像头**：2 × 5MP MIPI（RGB），建议补充广角与低光模块
* **麦克风**：4 麦克风阵列，支持声源定位与波束成形
* **扬声器**：2W ×2 立体声
* **显示屏**：5" IPS 触控，用于表情、设置与交互
* **传感器**：超声波避障、红外人体感应、温湿度、PM2.5（可选）
* **驱动**：双轮差分驱动，支持自主导航控制
* **电源**：18650 电池组，目标续航 6–8 小时（混合场景）

### 7.2 软件架构（模块化）

* **底层**：Ubuntu 22.04 + Robot OS（ROS 2）
* **感知层**：摄像头、麦克风、传感器驱动，数据预处理
* **推理层**：本地模型（人脸、目标检测、语音唤醒）+ 云模型（复杂语义）
* **服务层**：情感引擎、对话管理、任务调度、设备管理
* **接口层**：REST / gRPC 服务，MQTT（实时事件）、WebRTC（实时音视频）
* **前端**：iOS/Android App + Web 管理后台
* **云端**：模型更新仓库、数据分析、用户管理（可选）

### 7.3 网络架构

* 本地通信：Wi-Fi 6、Bluetooth 5.4
* 远程通信：HTTPS/TCP + WebRTC for AV
* 消息总线：MQTT（事件推送、轻量级控制）

## 8. 非功能性需求（NFR）

* **性能**：语音响应端到端 < 1.6 s（云回合），本地模型响应 < 1.2 s
* **可靠性**：MTBF（平均无故障时间）需满足家用品级别，年正常运行时间 ≥ 98%
* **安全性**：固件签名、TLS 1.2+/AES-256 加密、设备身份认证
* **隐私**：默认本地优先处理，人脸/情感数据默认存本地并加密；用户可选择同步云端
* **可维护性**：远程日志、自动健康上报、诊断工具
* **扩展性**：支持插件式技能市场与第三方 SDK

## 9. 隐私、安全与合规

### 9.1 隐私策略（核心点）

* 默认所有敏感生物识别数据（人脸模板、情感标签）保存在设备本地并加密；任何上报云端的个人数据必须经用户明确同意并在 App 中可撤销。
* 提供隐私模式：物理摄像头遮挡/麦克风静音/完全禁用录像的时间段设置。
* 记录与审计：所有重要访问（录像回放、远程查看）在 App 中记录审计日志并可导出。

### 9.2 安全机制

* 设备注册与身份：设备使用唯一证书进行注册与认证。
* OTA 签名与回退：固件与资源文件均需签名，支持失败回退。
* 最小权限原则：各模块采用沙箱策略，限制访问敏感接口。

### 9.3 合规要求

* 本地法规：PIPL（个人信息保护法）合规；儿童内容合规要求；电器/无线认证（CCC/CE/FCC）
* 安全合规：电池安全（UN38.3 测试）、电磁兼容、无线电发射合规

## 16. 风险评估与缓解措施

### 16.1 技术风险

* 模型在边缘设备效果不佳：缓解—使用云+边缘混合推理、模型量化与剪枝
* 自主导航/避障失败：缓解—限速运行、软限制区域、硬件碰撞传感器

### 16.2 市场风险

* 竞争对手价格战：缓解—强化差异化（情感陪伴、健康功能）、提升渠道与品牌合作

### 16.3 法律/合规风险

* 数据合规风险：缓解—默认本地优先、提供隐私白皮书与合规审计

### 16.4 供应链风险

* 芯片/模组断供：缓解—多供应商策略、关键件库存策略

## 17. 指标与度量（KPIs）

* 产品层：出货量、返修率、单机月活（MAU）
* 用户体验层：ASR 准确率、情绪识别准确率、视频延迟
* 运营层：客户满意度（CSAT）、首次响应时间、修复平均时间（MTTR）
* 商业层：ARPU（含订阅）、转化率（试用→付费）

## 18. 附录

### 18.1 建议 BOM（示例）

* RDK X5 主控模组 ×1
* 摄像头模组（5MP） ×2
* 麦克风阵列模组 ×1
* 5" 触摸屏 ×1
* 扬声器 ×2
* 电池组（18650 系列） ×1
* 底盘（差分驱动） ×1

### 18.2 API & 数据模型（概要）

* 设备注册 API：POST /api/v1/devices/register
* 远程视频：WebRTC session negotiation + TURN/STUN
* 告警事件：MQTT topic: /device/{id}/event
* 用户偏好：PUT /api/v1/users/{id}/preferences

### 18.3 用户界面与对话流（示例）

* 开机引导：设备开机 → 语音与屏幕引导配网 → App 绑定
* 用药提醒流程：系统提醒 → 用户语音/触摸确认 → 若无确认 3 次发送家属通知

### 18.4 核心语音指令集
